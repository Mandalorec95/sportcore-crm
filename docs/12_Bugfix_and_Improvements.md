# 12. Bugfix & Improvements Log

## Критические баги (исправлены)

### 1. Вкладка посещаемости — крэш при открытии
**Причина:** DTO ожидал `items[]`, а фронтенд отправлял `records[]`.
**Исправление:** `bulkMarkAttendance(sessionId, { items })` — ключ исправлен.
**Файл:** `frontend/app/attendance/page.tsx`

### 2. Перестановка firstName/lastName в карточке спортсмена
**Причина:** `fullName = "${firstName} ${lastName}"`, индекс `[0]` — имя, `[1]` — фамилия. Форма ред. использовала наоборот.
**Исправление:** Индексы переставлены.
**Файл:** `frontend/app/athletes/[id]/page.tsx`

### 3. Несоответствие полей при создании спортсмена
**Причина:** Фронтенд отправлял `fullName`, `dateOfBirth`, `sport`, а бекенд ожидал `firstName`, `lastName`, `birthDate`, `sportType`, `parent: {fullName, phone, email}`.
**Исправление:** Форма переписана, поля синхронизированы с DTO.
**Файл:** `frontend/app/athletes/page.tsx`

### 4. fullName не вычислялся в записях посещаемости
**Причина:** Бекенд возвращает `{ firstName, lastName }`, а не `fullName`.
**Исправление:** Добавлена функция `getAthleteFullName()`.
**Файл:** `frontend/app/attendance/page.tsx`

## Новые функции

### 5. Оценки (1–5) в журнале посещаемости
- Кнопки выбора оценки в строке каждого спортсмена
- Оценки сохраняются в базе (поле `grade Int?` в модели `Attendance`)
- Отображаются в родительском портале

### 6. CRUD соревнований
- Создание / редактирование / удаление соревнований
- Добавление результатов с выбором спортсмена, места, медали, дисциплины

### 7. Готовность к соревнованиям (цветовые индикаторы)
- Зелёный: посещ. ≥ 80% и оценка ≥ 4
- Жёлтый: посещ. 50–79% или оценка 3–3.9
- Красный: посещ. < 50% или оценка < 3
- Вкладка «Готовность» в детальном просмотре соревнования

### 8. Допуск к соревнованиям (согласование с родителями)
- Тренер: кнопки «Допустить» (запрос согласия), «Допустить без согласия», «Отклонить»
- Бекенд: модель `CompetitionApproval`, уведомление родителю
- Родитель: баннер с запросами + кнопки «Подтверждаю» / «Отказываю»

### 9. Tasks/Notes на странице уведомлений
- Две вкладки: «Уведомления» и «Задачи и заметки»
- CRUD задач (создание, статус, приоритет, срок)

### 10. Полный CRUD групп
- Создание, редактирование, удаление групп
- Назначение тренера, возрастной диапазон, вместимость, цена

### 11. Страница «Участники» (Users)
- Таблица всех пользователей с ролью и статусом
- Иконка замка — просмотр логина и демо-пароля
- Генерация нового пароля
- Создание / редактирование / удаление пользователей

### 12. Профиль администратора / тренера
- Просмотр и редактирование имени, email, телефона
- Смена пароля (с перенаправлением на login)

### 13. Автогенерация пароля при создании пользователя
- Хранится в поле `demoPassword` (шифруется bcrypt)
- Виден через иконку замка

### 14. Мобильная адаптивность
- Бургер-меню в сайдбаре
- Таблицы: `overflow-x-auto` + `min-w-[600px]`
- Отступы `p-4 md:p-6` на всех страницах

### 15. Оценки в родительском портале
- Средняя оценка в карточке ребёнка
- В таблице последних занятий — badge с оценкой
